# HTTP/1.0과 HTTP/1.1
  ```
  Connectionless 방식으로 동작
    - 연결을 유지하지 않는 통신방식
      클라이언트 -> 서버로 요청 후 서버 -> 클라이언트 응답 후 연결을 끊어버림  => 통신을 할 때마다 연결을 맺어야함
    - 요청이 간단하고 빠를 때 유용함 
    - Restful API에서 주로 사용하며 GET POST PATCH DELETE 등 메소드들 지원
    - 브라우저를 통해 데이터를 전송하며 다양한 디바이스에서 사용가능
    
    - 불필요한 데이터를 전송할 수 있어 대역폭 낭비가 발생이 가능하다.
    
  HTTP/1.1은 Connectionless에서 발전한 형태로 Keel-Alive 연결을 지원하여,
  여러 요청들을 하나의 요청으로 처리가 가능함
  
  * HTTPS
    -> HTTP 프로토콜에서 SSL이나 TLS 프로토콜을 함께 사용하여 보안연결 생성
       데이터의 암호화와 무결성을 유지
  ```

# WebSocket
  ```
  ConnectionFul 방식으로 동작
    1. 양방향 통신
      - 양방향 통신을 지원하는 프로토콜이며, 서버와 클라이언트간 지속적인 연결을 유지한다.
    2. 연결 유지
      - 서버와 클라이언트가 한번 연결되면 계속 연결을 유지하여 데이터를 주고받을 수 있다.
    3. 대역폭 절약
      - 웹소켓은 연결을 유지하며 통신을 하기 때문에, HTTP 통신과 달리 통신마다 연결을 맺을 필요가 없어 대역폭이 절약된다.
    4. 프로토콜의 다양성
      - 프로토콜을 독자가 직접 정의가 가능하여, 브라우저나 서버에 구애받지않고 다양한 플랫폼에서 사용 가능하다.
    5. 보안성
      - SSL과 TLS 프로토콜을 사용하여 보안연결을 통해 보안성을 높일 수 있다.
    - 실시간채팅, 주식, 게임 등 실시간 데이터가 필요한 경우에 주로 사용
    - HTML5에서 도입된 기술이며 TCP 기반의 프로토콜을 사용한다.
  ```
